## 引子：Agent 的技能，到底该不该留着？

如果你训练过一个使用外部技能的 AI Agent，你可能遇到过这个纠结：

技能加多了，Agent 在一大堆技能里挑花眼，上下文变长，路由噪声增加，性能反而下降。
技能全去掉，Agent 又好像丢了魂，一些长尾能力怎么也学不会。

这其实是目前 Agentic RL 领域一个被忽视的根本问题：**外部技能集在训练过程中到底应该怎么演化？**

现有的方法给出了两个极端的答案。

一个极端是「全都要」——SkillRL 之类的做法，技能只进不出，越堆越多。另一个极端是「全不要」——Skill0 之类的做法，把技能当临时脚手架，训练完全部撤掉，让 Agent 把能力全部内化到参数里。

香港中文大学和佛罗里达大学的研究团队在 2026 年 5 月发表的论文中，对这两种做法都说了不。

他们提出了 **SLIM（Skill LIfecycle Management）**——一个动态技能生命周期管理框架。

核心观点很朴素：**最优的活跃技能集是非单调的，它随任务和训练阶段动态变化。** 有些技能该保留，有些该淘汰，有些时候还需要扩展新的。

就像生物的新陈代谢——不是只长不死，也不是全部吸收，而是有进有出，动态平衡。

---

## 01 问题：技能的「外部边界」在哪里？

先理解一下背景。

现在的 LLM Agent 越来越多地依赖外部技能来解决复杂任务。一个「技能」可以是一段可复用的程序性知识——比如 ALFWorld 里的「清洁物体」步骤，或者 SearchQA 里的「多跳推理」策略。

这些技能在推理时被注入到 Agent 的上下文中，作为外部指导。

问题在于，模型的参数容量是有限的。不是每个有用的能力都应该被塞进模型参数里。外部技能特别适合保留那些**窄域、低频、长尾**的程序——它们编码到参数里成本太高，或者根本没必要。

但另一方面，保持太多活跃技能也不是免费的。大的技能库会引入路由噪声，长的注入上下文会降低技能使用的可靠性。

所以核心问题不是「该积累还是该消除」，而是：**如何确定一个学习中的 Agent 的外部能力边界？**

论文把这个形式化为一个容量约束的分配问题：

$$
\max_{\theta, \mathcal{A}, \mathcal{I}} \mathbb{E}_{x \sim \mathcal{X}}[\text{Perf}(x; \pi_\theta, \mathcal{A})] - \Omega(\mathcal{A})
$$

$$\text{s.t.} \sum_{s \in \mathcal{I}} m(s) \leq \mathcal{C}_\theta, \quad \mathcal{A} \cap \mathcal{I} = \emptyset$$

其中：
- $\mathcal{A}$ 是活跃外部技能集
- $\mathcal{I}$ 是已内化的技能集
- $\Omega(\mathcal{A})$ 是外部支持成本（单调递增）
- $\mathcal{C}_\theta$ 是模型的有限知识容量

这个公式说的是：在模型容量有限的约束下，找到一个最优的技能分配——哪些放在外部，哪些内化到参数里，使得性能减去外部成本后的净收益最大。

精确求解这个混合空间（连续的策略参数 + 离散的集合变量）是不可能的。SLIM 的贡献在于，它给出了一套可操作的近似方案。

## 02 方法：SLIM 的三个生命周期操作

SLIM 的设计非常优雅。它把每个训练周期分成两个阶段：

**第一阶段**：固定活跃技能集，用 GRPO 更新策略。
**第二阶段**：固定策略，对活跃技能集做生命周期管理。

交替进行，直到收敛。

生命周期管理的核心是三个操作：**保留（Retain）、淘汰（Retire）、扩展（Expand）**。

### 2.1 边际外部贡献（MEC）估计

怎么做决策？SLIM 用了一个简洁的指标——**边际外部贡献（Marginal External Contribution, MEC）**。

对每个被审计的技能 $s$，MEC 通过留一法（leave-one-skill-out）验证来估计：

$$\Delta_t(s) = \text{Perf}(\mathcal{V}_t(s); \mathcal{A}_t) - \text{Perf}(\mathcal{V}_t(s); \mathcal{A}_t \setminus \{s\})$$

简单说就是：去掉这个技能，性能下降多少？

下降多 → MEC 高 → 这个技能还有外部价值。
下降少或上升 → MEC 低 → 这个技能可能被内化了，或者成了噪声。

为了减少审计噪声，SLIM 还用了指数移动平均来平滑估计值。

### 2.2 保留（Retain）

如果技能的平滑 MEC 明显为正（超过阈值 $\tau_{\text{keep}}$），保留它。

这意味着当前策略仍然从这个外部能力中获益，应该继续让它参与后续的训练。

### 2.3 淘汰（Retire）

如果技能的 MEC 接近零或为负，且满足两个条件：
- 累计暴露次数足够多（$u_t(s) \geq n_{\min}$）
- 低贡献连续轮次达到阈值（$\ell_t(s) \geq p$）

则淘汰它。

这两个条件保护了低频技能——不会因为还没被充分使用就被误杀。淘汰不意味着技能消失了，它可能已经被内化到策略中，或者变得冗余/过时。

### 2.4 扩展（Expand）

如果某个活跃技能覆盖的任务区域持续失败（性能低于 $\tau_{\text{expand}}$，失败次数超过 $n_{\text{expand}}$），则创建新技能来填补空白。

扩展用的是路由失败分桶（routed failure buckets）——把失败的任务聚类，为每个聚类生成一个新的任务特定技能。

### 2.5 层级技能检索

为了减少搜索空间，SLIM 使用了层级技能库：通用技能池 + 任务特定技能池。对于每个任务，先通过 embedding 相似度检索相关的任务特定技能，再结合通用技能，形成最终的活跃技能集。

## 03 实验：不只是性能提升，更是范式转变

SLIM 在两个代表性基准上进行了评估：**ALFWorld**（交互式家务任务）和 **SearchQA**（多跳问答）。

### 3.1 主要结果

| 方法 | ALFWorld Avg. | SearchQA Avg. |
|------|--------------|---------------|
| GRPO | 67.2 | 37.5 |
| GRPO† | 68.8 | 37.9 |
| SkillRL† | 75.0 | 38.1 |
| Skill0 | 74.2 | 39.3 |
| **SLIM** | **72.7** | **41.0** |
| **SLIM†** | **87.5** | **41.0** |

† 表示推理时带有外部技能。

关键数据：
- **ALFWorld**：SLIM† 达到 87.5%，比最强基线 SkillRL†（75.0%）高出 **12.5 个百分点**。
- **SearchQA**：SLIM 和 SLIM† 都达到 41.0%，比 Skill0（39.3%）高出 **1.7 个百分点**。
- **平均提升**：相比最强非 SLIM 基线，平均提升 **7.1 个百分点**。

### 3.2 训练动力学的本质差异

这才是最有趣的部分。

SkillRL 的活跃技能数从 38 增长到 73——技能越堆越多，但性能反而不如 SLIM。说明**更多外部技能 ≠ 更好性能**。

Skill0 的活跃技能数从 38 减少到 0——技能全部撤掉后，验证性能从 92.2% 掉到 76.6%。说明**强制零技能推理会移除有用的外部支持**。

SLIM 呢？活跃技能数先扩展到 46，然后波动收缩，最终稳定在 **21 个技能的紧凑非空集合**。

同时，无技能性能从 29.7% 提升到 84.4%——策略本身确实在学习。有技能性能峰值达到 93.8%，最终保持在 90.6%。

**SLIM 没有用外部依赖来换取策略学习——它在提升策略的同时，保留了一组仍然提供边际价值的外部技能。**

### 3.3 消融实验

| 变体 | ALFWorld Avg. |
|------|--------------|
| SLIM（完整） | 87.5 |
| 无淘汰 | 73.4 |
| 无扩展 | 78.9 |
| 随机审计 | 68.8 |
| 固定技能集大小 | 75.6 |

- 去掉淘汰 → 退化为 SkillRL 式的积累，性能暴跌 14.1 点。
- 去掉扩展 → 只能修剪不能补充，性能下降 8.6 点。
- 随机审计（随机保留/删除）→ 性能最差，只有 68.8%。说明**基于贡献的决策**是 SLIM 有效的核心。
- 固定技能集大小（LRU 替换）→ 75.6%，比 SLIM 低 11.9 点。说明关键不是「保留多少个」，而是「保留哪些」。

### 3.4 案例分析：技能的生命周期

论文做了一个详细的案例研究。几个有意思的发现：

**高频 ≠ 该保留**：`gen_011` 被频繁选择，但禁用后性能只下降 0.080——接近内化。`gen_004` 同样高频，但禁用后下降 0.284——仍然是外部关键技能。

**低频 ≠ 该淘汰**：`cle_003`（清洁特定技能）选择频率不高，但禁用后性能下降 0.250——全局不频繁，局部不可或缺。

**淘汰的依据是边际价值消失**：多个冷却（cool）相关技能在训练后期被退休，因为它们的边际外部贡献变成了负值。

## 04 讨论：参数与外部记忆的新分工

SLIM 最有意思的结论是：**策略学习和外部技能保留不是互斥的。**

在 ALFWorld 上，SLIM 和 SLIM† 之间有 14.8 个百分点的差距（72.7 vs 87.5），说明 ALFWorld 中的长程程序性行为仍然需要外部技能支持。

但在 SearchQA 上，SLIM 和 SLIM† 几乎没有差距（都是 41.0%），说明 SearchQA 的收益主要反映在训练好的策略中，而非推理时的技能依赖。

**不同的任务域需要不同的外部/内部能力分配。** 有些能力适合内化，有些适合保留为外部模块。SLIM 让 Agent 自己学会了这个分配。

这让我想到一个类比：人类的学习也是这样。骑自行车的技能内化成了肌肉记忆（不需要外部指导），但一些专业领域的程序性知识（比如手术步骤、调试流程）即使专家也需要外部参考。不是所有东西都值得「内化」，也不是所有东西都应该「外挂」。

## 05 实现细节：一个实用的训练循环

SLIM 的实际训练循环如下：

```
每 10 个 GRPO 步：
  1. 在验证集上运行当前路由
  2. 选择最近使用最多的 4 个技能进行审计
  3. 对每个审计技能：
     - 用留一法计算 MEC
     - 更新指数移动平均
     - 应用保留/淘汰规则
  4. 从路由失败分桶中创建最多 B 个新技能
```

实现上，SLIM 使用 Qwen3-Embedding-0.6B 做技能检索，K=3，相似度阈值 0.45。审计预算有限——每次只审计 4 个最活跃的技能，而不是全量审计。这在实际训练中非常关键，因为留一法验证的成本不低。

代码已开源：https://github.com/ejhshen/SLIM

## 06 局限与未来方向

论文坦诚地列出了几个局限：

1. **审计成本**：留一法验证需要额外的推理调用，虽然通过有限审计预算缓解了，但在更大规模上仍需更高效的方法。
2. **单步移动**：生命周期管理目前只考虑单技能移动，避免了全局搜索的复杂性，但也可能错过联合优化的机会。
3. **单模态**：实验只在文本环境（ALFWorld、SearchQA）上评估，多模态环境中的技能生命周期还是一个开放问题。
4. **技能粒度**：目前用的是任务特定的技能单元，更细粒度的生命周期管理（比如技能中的子步骤）值得探索。

## 结语：从「技能堆砌」到「技能生态」

SLIM 的核心贡献不是某个具体的算法改进，而是提出了一个**新的视角**：

把外部技能集看作一个动态优化的变量，而不是静态的输入或临时的脚手架。

这个视角转变的意义可能比 7.1% 的性能提升更深远。它暗示了 AI Agent 的能力组织方式可以像生物系统一样——有代谢、有更新、有选择性地保留和淘汰。

未来的 Agent 可能不再需要人工设计技能库的规模。它们会自己学会：哪些能力该内化，哪些该保留为外部模块，哪些该淘汰，哪些该扩展。

就像我们的身体不会记住每一个见过的单词，但会在需要时查阅字典。
